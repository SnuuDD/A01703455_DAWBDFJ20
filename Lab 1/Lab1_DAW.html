URL

Las URLs se utilizan para localizar recursos, porporcionando una identificacion a la direccion del recurso. Una vez encontrando dicho recursos, el sistema realiza una serie de operaciones en dicho recurso, normalmente identificadas con las palabras 'access', 'update', 'replace', 'find attributes'. En general, solo el metodo 'access' debe especificarse para cualquier esquema de URL. [1]

MIME

Multipurpose Internet Mail Extensions (MIME) es un estándar de Internet para describir los tipos de contenido en los mensajes. Los mensajes MIME pueden contener texto, imágenes, audio, video y otros datos específicos de aplicación. La información oficial de MIME es proporcionada por Internet Engineering Task Force (IETF). [2]

HTTP

Hypertext Transfer Protocol (HTTP) es un protocolo de nivel de aplicación para sistemas de información distribuidos, colaborativos y de hipermedia. Es un protocolo genérico, sin estado y orientado a objetos que se puede utilizar para muchas tareas, como nombres de servidores y sistemas de gestión de objetos distribuidos, a través de la extensión de sus métodos de solicitud. [3]

HTML5

HTML5 (HyperText Markup Language, versión 5) es la quinta revisión del ya existente HTML el cual establece una serie de nuevos elementos y atributos que reflejan el uso típico de los sitios web modernos. [4]

JS

JavaScript (JS) es un lenguaje ligero e interpretado, orientado a objetos con funciones de primera clase, es un lenguaje script multi-paradigma, basado en prototipos, dinámico, soporta estilos de programación funcional, orientada a objetos e imperativa. [4]

PHP

PHP es un lenguaje interpretado del lado del servidor que se caracteriza por su potencia robustez y modularidad, los programas estan embebidos directamente en el codigo HTML y se ejecutan desde el servidor web. [5]

JSP

Java Server Pages (JSP) es una extensión de la plataforma Java que promete convertirse en la herramienta estándar para el desarrollo de aplicaciones web. Java Server Pages permite a los desarrolladores crear fácilmente páginas web dinámicas e interactivas incorporando scripts directamente en HTML y XML. [6]

AJAX

JavaScript Asíncrono + XML (AJAX) no es una tecnología por sí misma, es un término que describe un nuevo modo de utilizar conjuntamente varias tecnologías existentes. Esto incluye: HTML o XHTML, CSS, JavaScript, DOM, XML, XSLT, y XMLHttpRequest. Gracias a esta tecnologia es posible lograr aplicaciones web capaces de actualizarse continuamente sin tener que volver a cargar la página completa. [7]



¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet se le conoce como un conjunto descentralizado de redes de comunicacion regidos por los protocolos TCP/IP. En cambio WWW por sus siglas en ingles (World Wide Web) es un sistema de distribucion de documentos de hipertexto e hipermedia que se rige por diferentes estandares, entre ellos: URL, HTTP, HTML, XML entre otros.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

Para realizar solicitudes a un servidor por medio del protocolo HTTP, se utilizan metodos predefinidos del mismo protocolo que indican que acciones se deber realizar sobre los recursos.
  • GET: El método GET solicita una representación de un recurso específico. Las peticiones que usan el método GET sólo deben recuperar datos
  • HEAD: El método HEAD pide una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta.
  • POST: El método POST se utiliza para enviar una entidad a un recurso en específico, causando a menudo un cambio en el estado o efectos secundarios en el servidor.
  • PUT: El modo PUT reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición.
  • PATCH: El método PATCH es utilizado para aplicar modificaciones parciales a un recurso.
  • DELETE: El método DELETE borra un recurso en específico.
[8]

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Ya que al enviar un formulario proporcionamos un bloque de datos que se deben procesar utilizamos el método POST ya qye solicita que el recurso de destino procese la representación incluida en la solicitud de acuerdo con el recurso en una semántica propia y específica.[8]

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

El método GET solicita la transferencia de la actual representación seleccionada del recurso de destino. GET es el mecanismo principal de recuperación de información y el foco de casi todas las optimizaciones de rendimiento. Por lo tanto, cuando las personas hablan de recuperar información identificable a través de HTTP, generalmente se refieren a hacer una solicitud GET.[10]

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

El código de estado 200 (OK) indica que la solicitud ha tenido éxito.[8]

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Si, debido a que el codigo de estado 404 (Not Found) indica que el servidor de origen no encontró una representación actual para el recurso de destino al que e quiere acceder o no está dispuesto a revelar que existe.[8]

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

No necesariamente, ya que el código de estado 500 (Error interno del servidor) indica que el servidor encontró una condición inesperada que le impidió cumplir la solicitud. [8]

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados?

Con la llegada de HTML5, se han desaprobado muchos elementos, debido a que estos han sido remplazados por nuevas etiquetas, funciones de CSS, o simplemente ya no se utilizan.
  • Appet: Que en su lugar se utiliza "embed" u "object"
  • Acronym: Que en su lugar se utiliza "abbr"
  • Frame: Estos han sido completamente desaprobados en HTML5
[9]

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

  • HTML5 define una sintaxis compatible con HTML4 y XHTML 1.0.
  • Para definir el juego de caracteres se introduce un nuevo atributo para la etiqueta "meta": "meta charset=”UTF-8″"
  • Simplificacion del DOCTYPE: "!DOCTYPE html"
  • HTML5 permite incluir elementos de SVG y MathML.
  • Se introducen nuevos elementos, como: section, article, aside, header, footer, etc.
[10]

¿Qué componentes de estructura y estilo tiene una tabla?

Una tabla en HTML se define con la etiqueta "table".
El encabezado de tabla se define con la etiqueta "th". Por defecto, los encabezados de las tablas están en negrita y centrados.
Cada fila de la tabla se define con la etiqueta "tr".
Las celdas de la tabla se definen con la etiqueta "td".

¿Cuáles son los principales controles de una forma HTML5?

  • "!DOCTYPE HTML": Ya qye define que el documento esta bajo el estandar de HTML 5
  • "HTML" Represneta la raiz del documento.
  • head" Representa una colección de metadatos acerca del documento, scripts y hojas de estilo.
  • "title" Define el título del documento, el cual se muestra en la barra de título del navegador o en las pestañas de página.
  • "link" Usada para enlazar JavaScript y CSS externos con el documento HTML actual.
  • "body" Representa el contenido principal de un documento HTML.
  • "header" Define la cabecera de una página o sección
  • "footer" Define el pie de una página o sección.
  • "main" Define el contenido principal o importante en el documento.
[11]
